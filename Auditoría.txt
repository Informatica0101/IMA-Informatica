AUDITORÍA DE ARCHIVOS - PRESENTACIONES
=======================================

Archivo: Informatica_I/medidas_navegacion_extensiones.html
- Encabezados/Pies: Usa #main-header, #mobile-header, #main-footer.
- Lógica FS: handleFullscreenChange() intenta ocultar pero usa selectores individuales.
- Fuentes actuales: h1: 2em, h2: 1.6em, h3: 1.2em, p/ul/ol: 0.8em.

Archivo: Informatica_I/Partes_Computadora.html
- Encabezados/Pies: Usa .xp-taskbar, .logo-fixed. (Carga estilos1.css)
- Lógica FS: handleFullscreenChange() presente pero incompleta para elementos XP.
- Fuentes actuales (estilos1.css): h2: 2.2rem, h3: 1.5rem, p/ul/ol: 1.1rem.

Archivo: Informatica_I/hojas_calculo.html
- Encabezados/Pies: Usa #page-header, #page-footer.
- Lógica FS: Usa body.is-fullscreen CSS y handleFullscreenChange() JS.
- Fuentes actuales: @media (max-width: 768px) reduce h2/h3 a 1.4rem, p/li a 0.9rem. Base no especificada explícitamente en rem para todos.

Archivo: III_BTP_A/dw_II/fundamentos_js.html
- Encabezados/Pies: Usa .logo-fixed.
- Lógica FS: handleFullscreenChange() básica.
- Fuentes actuales: h2: 2.2rem, h3: 1.6rem, p/ul/ol: 1.05rem.

Archivo: II_BTP_A/Programacion/introduccion_programacion.html
- Encabezados/Pies: Usa #page-header, #page-footer.
- Lógica FS: handleFullscreenChange() básica.
- Fuentes actuales: h2/h3: 1.4rem (mobile), p/li: 0.9rem (mobile).

Hallazgos Generales:
- La lógica de pantalla completa no es exhaustiva; a menudo olvida logos fijos o barras de tareas específicas.
- El uso de .style.display = 'none' es inconsistente entre archivos.
- Las fuentes son pequeñas en móviles, dificultando la lectura en modo presentación.
