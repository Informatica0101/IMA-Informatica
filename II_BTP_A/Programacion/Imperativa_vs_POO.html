Evolución de los Paradigmas

A medida que los problemas de software se volvieron más complejos, la forma de escribir código tuvo que evolucionar para permitir el mantenimiento y la reutilización.

Un paradigma de programación es un estilo o "forma de pensar" sobre cómo organizar el código para resolver un problema.

Programación Imperativa

Se basa en dar órdenes directas a la computadora sobre cómo debe realizar cada paso.

El código es una secuencia de instrucciones que modifican el estado del programa (el valor de sus variables) hasta llegar a un resultado.

Enfoque en el Procedimiento

En este modelo, las funciones y procedimientos son el centro de atención. Los datos son "pasados" de una función a otra.

  • Lógica separada de los datos.
  • Ejecución lineal y secuencial.
  • Dificultad para escalar en sistemas muy grandes.

Programación Orientada a Objetos (POO)

Este paradigma organiza el software alrededor de "objetos" en lugar de "acciones" y datos en lugar de "lógica".

Un objeto es una entidad que combina datos (atributos) y comportamientos (métodos) en una sola unidad lógica.

La Clase: El Plano de Construcción

Para crear objetos, primero necesitamos una Clase.

Es una plantilla o molde que define qué propiedades y qué acciones tendrán los objetos que se creen a partir de ella.

Ejemplo: La clase "Automóvil" define que todos los carros tienen color y pueden acelerar.

Los Objetos: Instancias Reales

Cuando usamos una clase para crear un elemento concreto, estamos haciendo una instancia.

Si "Automóvil" es la clase, un "Toyota Rojo" específico es el objeto. Cada objeto tiene sus propios valores para los atributos definidos en la clase.

Atributos y Métodos

Son los dos componentes fundamentales que definen a un objeto en la POO.

Atributos (Datos):

Características o propiedades.
(Ej: nombre, edad, saldo, color).

Métodos (Acciones):

Funciones que el objeto puede realizar.
(Ej: caminar, depositar, frenar).

Ventaja: Encapsulamiento

Protege los datos internos de un objeto, permitiendo el acceso solo a través de métodos seguros.

Evita que otras partes del programa modifiquen los datos de forma accidental o incorrecta, aumentando la seguridad y estabilidad.

Ventaja: Abstracción

Nos permite manejar la complejidad ocultando los detalles innecesarios y mostrando solo lo relevante.

Para usar un control remoto, no necesitas saber cómo funciona la circuitería interna; solo necesitas saber qué botones presionar.

Resumen Comparativo

Característica Imperativa POO
Enfoque Algoritmos / Pasos Objetos / Entidades
