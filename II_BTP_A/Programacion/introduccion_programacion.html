Introducción a la Programación

Módulo II: Lógica de Programación

Lic. Pablo Antonio Peña Mancia

Fecha: 1 de febrero de 2026

Grado: Undécimo de Informática

1. Lógica de Programación

La lógica de programación es la organización coherente de las instrucciones necesarias para que una computadora ejecute una tarea. Es la base indispensable antes de escribir código en cualquier lenguaje.

2. Importancia de la Lógica

Sin lógica, los programas serían una serie de órdenes desordenadas que la computadora no entendería o ejecutaría erróneamente.

Claridad

Permite que el código sea fácil de leer y mantener.

Optimización

Ayuda a encontrar la solución más rápida con menos recursos.

3. ¿Qué es la Programación?

Es el arte de convertir un algoritmo en un lenguaje que la computadora pueda entender. Implica diseñar, escribir, probar y mantener el código fuente de un programa informático.

4. Lenguajes de Programación

Son sistemas de símbolos y reglas que permiten al programador comunicarse con el hardware de la computadora.

Ejemplos populares: Python, Java, C++, C#, JavaScript y PHP.

5. Clasificación de Lenguajes

Lenguaje de Máquina:

Código binario (0 y 1) que el procesador entiende directamente.

Bajo Nivel (Ensamblador):

Usa mnemotécnicos cercanos a la arquitectura del hardware.

Alto Nivel:

Cercanos al lenguaje humano, permitiendo portabilidad y facilidad de uso.

6. Compiladores e Intérpretes

Para que la computadora ejecute el código de alto nivel, necesita traducirlo.

  • Compilador: Traduce todo el programa de una vez y genera un archivo ejecutable (.exe).
  • Intérprete: Traduce y ejecuta el código línea por línea en tiempo real.

7. El Ciclo de Vida del Software

Es el proceso estructurado que se sigue para desarrollar software de calidad, desde la idea inicial hasta el retiro del producto.

8. Fase 1: Análisis del Problema

Consiste en comprender exactamente qué debe hacer el programa. Se definen las entradas, los procesos y las salidas esperadas.

Pregunta clave: ¿Qué es lo que se quiere resolver?

9. Fase 2: Diseño del Algoritmo

Se crea un plan detallado paso a paso para resolver el problema analizado anteriormente.

10. Fase 3: Codificación

Es la traducción del algoritmo a un lenguaje de programación específico (escritura del código fuente).

print("Escribiendo el código...");

11. Fase 4: Pruebas y Depuración

Se ejecuta el programa con diferentes datos para encontrar errores (bugs) y corregirlos antes de la entrega final.

12. ¿Qué es un Algoritmo?

Es una serie de instrucciones finitas, precisas y ordenadas para alcanzar un objetivo o solucionar un problema.

"Un algoritmo es la receta, el programa es el plato terminado".

13. Características de los Algoritmos

  • Preciso: Debe indicar el orden exacto de realización de cada paso.
  • Definido: Si se sigue dos veces con los mismos datos, se obtiene el mismo resultado.
  • Finito: Debe tener un número determinado de pasos y terminar.

14. Partes de un Algoritmo

Entrada: Información necesaria para comenzar.
Proceso: Operaciones y cálculos a realizar.
Salida: Resultado final obtenido.

15. Técnicas de Representación

Los algoritmos se pueden representar de varias formas antes de programarlos:

Pseudocódigo: Lenguaje narrativo técnico que simula la programación.
Diagrama de Flujo: Representación gráfica mediante símbolos normalizados.

16. Estructuras de Control

Controlan el flujo de ejecución de las instrucciones:

  • Secuenciales: Un paso tras otro.
  • Condicionales: Toman decisiones (Si/Entonces).
  • Cíclicas: Repiten instrucciones (Mientras/Para).
