¿Qué es un Diagrama de Flujo?

Es una representación gráfica de un algoritmo. Muestra visualmente los pasos que sigue un programa y cómo fluye la información de un punto a otro.

"Una imagen vale más que mil líneas de código".

Simbología Básica

Terminal

Inicio o Fin del algoritmo.

Proceso

Cálculos o acciones directas.

Símbolos de Interacción

Entrada / Salida

Lectura de datos o impresión de resultados.

Decisión

Condición SI/NO (Verdadero/Falso).

Líneas de Flujo

Son flechas que indican la dirección en la que se ejecutan las instrucciones. El flujo SIEMPRE debe ir de arriba hacia abajo o de izquierda a derecha.

Reglas de Construcción

  • Todo diagrama debe tener un Inicio y un Fin.
  • Las líneas de flujo deben ser rectas.
  • No pueden quedar líneas "sueltas" o sin conectar.
  • De un rombo de decisión SIEMPRE deben salir dos caminos (Sí/No).

Ejercicio Práctico 1

Dibuja el diagrama de flujo para un algoritmo que pida un número y muestre en pantalla si es positivo (Mayor a 0).

Estructuras de Control

Secuencial
Selectiva (SI-SINO)
Repetitiva (Ciclos)

Conectores

Se usan círculos pequeños para unir partes del diagrama cuando no caben en una sola página o para evitar que las líneas se crucen demasiado.

Ventajas Visuales

  • Facilita la comunicación con el cliente.
  • Permite detectar bucles infinitos visualmente.
  • Estandariza la lógica del equipo de desarrollo.

Software de Diseño

Herramientas para crear diagramas profesionales:

PSeInt
Lucidchart
Draw.io

Análisis de Errores

"Un diagrama con líneas que se cruzan o sin fin definido es un programa que fallará."

  IF condicion == TRUE THEN

    GOTO ETIQUETA_INICIO_CICLO

  ELSE

    TERMINAR()

  ENDIF
