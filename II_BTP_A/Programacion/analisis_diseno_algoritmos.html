Paso 1: Análisis

El análisis consiste en estudiar el problema para determinar QUÉ es lo que se requiere.
Se centra en las necesidades, no en el código.

Componentes del Análisis

Datos de Entrada

Valores que el usuario o el sistema proporcionarán.

Proceso

Transformación de datos (fórmulas, lógica).

Resultados (Salida)

Información final que el usuario espera.

Ejemplo de Análisis

Problema: Calcular el área de un círculo.

  • Entrada: Radio (r).
  • Proceso: Area = PI * r * r.
  • Salida: El valor del Area.

Paso 2: Diseño

El diseño define CÓMO el programa realizará la tarea. Es la creación de la lógica paso a paso.

Herramientas: Lenguaje Natural, Diagramas de Flujo y Pseudocódigo.

Lenguaje Natural

Descripción de los pasos en nuestro idioma cotidiano (español), sin reglas de código, pero manteniendo el orden lógico.

Ejercicio Práctico 1

Realiza el análisis (E-P-S) y el diseño en lenguaje natural para un algoritmo que determine si una persona es mayor de edad.

Refinamiento por Pasos

También llamado Diseño Descendente. Consiste en dividir un paso general en pasos más específicos hasta llegar a instrucciones simples.

No Ambigüedad

Cada instrucción en el diseño debe ser clara.
Evita términos vagos como "un poco", "quizás" o "después".

Incorrecto: "Suma los números"
Correcto: "Resultado = A + B"

Modularidad

Divide la lógica en bloques independientes llamados módulos o funciones para facilitar el mantenimiento.

Verificación de Lógica

Antes de programar, recorre mentalmente tus pasos de diseño (Prueba de Escritorio manual) para detectar errores antes de tocar la PC.

La Regla de Oro

"Si no puedes explicarlo paso a paso en una hoja de papel, no podrás programarlo en una computadora."

Lógica de Decisión

// Diseño de Comparación

LEER edad

IF edad >= 18 THEN

  ESCRIBIR "Es Mayor"

ELSE

  ESCRIBIR "Es Menor"

ENDIF
