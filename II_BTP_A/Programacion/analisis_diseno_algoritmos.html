Fase de Análisis

Consiste en comprender profundamente el problema.
Preguntas clave:
1. ¿Qué datos de entrada tengo?
2. ¿Qué resultados se esperan (salida)?
3. ¿Qué procesos debo realizar?

Definición de Requisitos

Los requisitos son las condiciones que el algoritmo debe cumplir.
Ejemplo: "El programa debe validar que la edad sea mayor a cero".

Fase de Diseño

Es la creación del plano o boceto de la solución. No usamos código real todavía, sino herramientas de representación.

Eficiencia de Algoritmos

Un buen algoritmo no solo resuelve el problema, sino que lo hace usando la menor cantidad de tiempo y memoria posible.

Prueba de Escritorio

Es el seguimiento manual de un algoritmo con datos reales para verificar si los resultados son los esperados antes de programar.

Ejercicio Práctico

Realizar la prueba de escritorio para un algoritmo que calcula el promedio de 3 notas: 70, 80 y 90. ¿Cuál es el resultado esperado?

Estructuras de Control

Los algoritmos usan caminos lógicos:

  • Secuencial: Uno tras otro.
  • Selectiva: Decisiones (Si / Entonces).
  • Repetitiva: Bucles (Mientras / Para).

Diagrama de Flujo

Representación gráfica de un algoritmo. Usa símbolos estándar como óvalos (Inicio/Fin), rombos (Decisión) y rectángulos (Proceso).

Pseudocódigo

Es un "falso lenguaje" escrito en español que permite enfocarse en la lógica sin preocuparse por las reglas estrictas de un lenguaje de programación real.

Mantenibilidad

El diseño debe ser claro para que otros programadores (o tú mismo en el futuro) puedan entenderlo y modificarlo fácilmente.

Robustez

Un algoritmo es robusto si es capaz de manejar errores de entrada (como que el usuario escriba letras donde deben ir números) sin detenerse bruscamente.

Lógica de Diseño

// Algoritmo Mayor de 2 números

IF A > B THEN

  PRINT "El mayor es A"

ELSE

  PRINT "El mayor es B"

ENDIF
