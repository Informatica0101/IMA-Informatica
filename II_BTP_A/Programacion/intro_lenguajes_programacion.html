El Concepto de Algoritmo

Decir que un problema se puede resolver algorítmicamente implica que es posible escribir un programa que producirá la respuesta correcta.

Definición: Conjunto claramente especificado de instrucciones sencillas a seguir para resolver un problema o calcular una función.

Teoría de la Computabilidad

Nació en la década de 1930 para formalizar el estudio de los algoritmos antes de que existieran las computadoras físicas.

  • Se enfoca en caracterizar los problemas que tienen solución algorítmica.
  • Presenta problemas que no pueden ser resueltos por máquinas.

Menciones Históricas

Alan Turing (1912-1954)

Demostró la insolubilidad del "Problema del Paro" y diseñó los primeros computadores electrónicos programables.

Kasparov vs Deep Blue

En 1997, la supercomputadora de IBM derrotó al campeón mundial de ajedrez (3.5 a 2.5), marcando un hito en la IA.

El Problema del Paro

Es el desafío de determinar si un programa llegará a un punto de detención o entrará en un ciclo infinito con una entrada dada.

Resultado de Turing:

No puede existir un programa de computadora que resuelva este problema para cualquier algoritmo arbitrario.

Complejidad Computacional

Estudio de las necesidades de tiempo y espacio (memoria) que requiere un programa para ejecutarse.

  • Un problema puede ser resoluble en teoría, pero no en la práctica por requerir siglos de ejecución.
  • Se mide en número de instrucciones o bits de almacenamiento.

Lenguajes de Alto Nivel

Permiten a los programadores escribir instrucciones usando palabras similares al lenguaje humano (como el inglés) y símbolos matemáticos.

Ventaja: Son independientes de la arquitectura de la computadora (portables).

Lenguajes Compilados

El código fuente se traduce íntegramente a lenguaje de máquina antes de ser ejecutado.

  • Proceso: El compilador genera un archivo ejecutable (.exe, .bin).
  • Velocidad: Son mucho más rápidos durante la ejecución de producción.

Lenguajes Interpretados

Un programa llamado intérprete lee y ejecuta las instrucciones una por una en tiempo real.

Diferencia: No generan un archivo ejecutable previo; la traducción ocurre mientras el programa corre.

Lenguajes Declarativos

En este enfoque, el programador le indica a la computadora qué desea obtener, sin especificar cómo obtenerlo.

Ejemplo: SQL para bases de datos.

Se describe el problema en lugar de proveer los pasos detallados de la solución.

Características del Enfoque Declarativo

  • Concentración en la lógica, no en la secuencia de control.
  • Las variables suelen ser inmutables (un solo valor).
  • El orden de ejecución no afecta el resultado final.
  • Las listas son la estructura fundamental de datos.

Ventajas y Desventajas

Ventajas:

Código más corto, fácil de mantener y razonar matemáticamente.

Desventajas:

Menos eficiente en uso de memoria y no apto para todos los problemas.

Programación Lógica

Basada en la lógica de predicados. El programa es un conjunto de hechos y reglas.

Utiliza mecanismos de inferencia para responder consultas a partir de los datos proporcionados.

Programación orientada a Bases de Datos

Sistemas diseñados para gestionar grandes volúmenes de información de forma estructurada.

  • SQL: Lenguaje estándar para interactuar con bases de datos relacionales.
  • Permite crear, leer, actualizar y borrar (CRUD) datos eficientemente.
