Programación Lógica

Consiste en emplear la lógica como lenguaje de programación. Es declarativa porque sirve para especificar qué problema resolver (condiciones).

Se especifican las condiciones que satisfacen las soluciones, y estas se deducen a partir de las mismas.

Relaciones y Entidades

En la programación lógica se trabaja de forma descriptiva, estableciendo relaciones entre entidades.

Relación: El vínculo o propiedad que une a los elementos.
Entidad (Objeto): Los elementos o sujetos sobre los que recae la relación.

Idea Esencial

Algoritmos = Lógica + Control

  • Lógica: El conocimiento especificado en un lenguaje formal.
  • Control: Mecanismo de inferencia que actúa sobre el conocimiento.

Ejemplo de Aplicación

Si queremos representar que "Juan es el padre de María":

padre(juan, maria).

Relación: padre

Objetos (Entidades): juan, maria

Prolog

Es uno de los lenguajes más representativos. Clásico de la Inteligencia Artificial y aplicado en el desarrollo de software comercial.

Utiliza la lógica de primer orden para resolver problemas mediante un mecanismo de inferencia.

El Bucle "Repetir"

Es una estructura iterativa que ejecuta un bloque de código y luego evalúa una condición.

Diferencia clave: A diferencia del "Mientras", el bloque de instrucciones se ejecuta al menos una vez.

Sintaxis en Pseudocódigo

Repetir
  Instrucciones...
Hasta Que (condición)

Se repite el proceso MIENTRAS la condición sea falsa, y se sale cuando sea verdadera.

Flujo de Ejecución

  1. Se ejecutan las instrucciones una vez.
  2. Se evalúa la condición al final.
  3. Si es Falsa, vuelve al paso 1.
  4. Si es Verdadera, termina el bucle.

Ejemplo: Validar número positivo

Repetir
  Escribir "Ingrese un número mayor a 0:"
  Leer n
Hasta Que (n > 0)

El programa seguirá pidiendo el número hasta que el usuario ingrese uno correcto.

Mientras vs Repetir

Mientras:

  • Evalúa al inicio.
  • Puede ejecutarse 0 veces.

Repetir:

  • Evalúa al final.
  • Se ejecuta al menos 1 vez.
