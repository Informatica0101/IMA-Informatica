Programación Lógica

Es un paradigma basado en la lógica de predicados. En lugar de dar órdenes, definimos Hechos y Reglas, y el sistema deduce la respuesta.

Lenguaje representativo: PROLOG.

Hechos y Reglas

Hecho

"Juan es padre de Pedro". (Afirmación siempre verdadera).

Regla

"X es abuelo de Y si X es padre de Z y Z es padre de Y".

El Bucle REPETIR (DO-UNTIL)

Repite un bloque de código HASTA QUE una condición se cumpla (sea verdadera). A diferencia del Mientras, este se ejecuta al menos una vez.

Estructura del REPETIR

REPETIR

  // Instrucciones (Se ejecutan primero)

HASTA QUE condicion == VERDADERA

Diferencia con Mientras

Mientras (While)

Evalúa ANTES de entrar. Puede no ejecutarse nunca.

Repetir (Do-Until)

Evalúa DESPUÉS. Se ejecuta 1 vez mínimo.

Sistemas Expertos

La programación lógica es la base de los Sistemas Expertos, programas que imitan el razonamiento de un especialista humano en un área específica (ej. Medicina).

Bases de Conocimiento

En lugar de bases de datos simples, la programación lógica usa Bases de Conocimiento donde la información está interrelacionada lógicamente.

Uso del Repetir en Menús

Es el bucle perfecto para crear menús de usuario que se muestran una vez y se repiten hasta elegir "Salir".

REPETIR

  MOSTRAR_MENU()

  LEER opcion

HASTA QUE opcion == 4 // 4 es Salir

IA y Lógica

La lógica es fundamental para el desarrollo de la Inteligencia Artificial simbólica, permitiendo que las máquinas "entiendan" relaciones de causa y efecto.

Pensamiento Deductivo

"Programar con lógica es enseñar a la computadora a conectar los puntos por sí misma basándose en la verdad de los datos."

Ejercicio Avanzado

Validación con Repetir:

REPETIR

  ESCRIBIR "Ingrese nota (0-100):"

  LEER nota

  IF nota < 0 OR nota > 100 THEN

    PRINT "ERROR: Fuera de rango"

  ENDIF

HASTA QUE nota >= 0 AND nota <= 100
