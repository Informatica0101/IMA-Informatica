Lenguajes Declarativos

Son aquellos en los cuales se le indica a la computadora qué es lo que se desea obtener o qué es lo que se está buscando.

En lugar de proveer una solución paso a paso (el "cómo"), el programador describe el problema y deja la interpretación a un motor de ejecución.

Características Clave

  • Orientación a resultados: Foco en la lógica del algoritmo, no en el control de la secuencia.
  • Transparencia referencial: Las variables suelen tener un solo valor asignado (sin asignación destructiva).
  • Inmutabilidad: El orden de ejecución no suele afectar el resultado final.

Ejemplos Comunes

SQL

Usado para describir qué datos extraer de una base de datos (SELECT nombre FROM empleados...).

HTML

Describe la estructura de una página web (qué elementos hay) sin detallar cómo dibujarlos píxel por píxel.

Balance del Paradigma

Ventajas:

  • Alto nivel de abstracción.
  • Soluciones fáciles de entender por humanos.
  • Ideal para Inteligencia Artificial y Bases de Datos.

Desventajas:

  • Menos eficiente (fase de interpretación extra).
  • Restringido a problemas específicos.

Declarativo vs Imperativo

Mientras el imperativo dice "Camina 10 pasos, dobla a la derecha, camina 5 pasos", el declarativo dice:

"Quiero estar en la panadería".

El "cómo" llegar queda a cargo del sistema subyacente.

El Bucle "Mientras" (While)

Es una estructura que repite un bloque de instrucciones mientras una condición sea verdadera.

La condición se evalúa antes de ejecutar el cuerpo del bucle. Si es falsa desde el inicio, el bucle nunca se ejecuta.

Sintaxis en Pseudocódigo

Mientras (condición) Hacer
  Instrucciones...
FinMientras

Es vital que dentro de las instrucciones algo cambie la condición para evitar un bucle infinito.

Flujo de Ejecución

  1. Se evalúa la condición lógica.
  2. Si es Verdadera, se ejecutan las instrucciones y se vuelve al paso 1.
  3. Si es Falsa, se salta el bucle y continúa el programa.

Ejemplo: Contador hasta 5

i = 1
Mientras (i <= 5) Hacer
  Escribir i
  i = i + 1
FinMientras

Salida esperada:

1, 2, 3, 4, 5

¿Cuándo usar "Mientras"?

  • Cuando no sabemos cuántas veces se repetirá el proceso (ej: esperar a que el usuario presione una tecla).
  • Para validar entradas de datos.
  • Para procesos que dependen de estados externos (ej: lectura de archivos).
