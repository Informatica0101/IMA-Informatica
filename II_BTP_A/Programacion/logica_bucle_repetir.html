Programación Lógica

Es un paradigma de programación basado en la lógica formal, específicamente en la lógica de predicados.

En este modelo, no decimos a la computadora "cómo" hacer las cosas, sino que definimos relaciones y permitimos que el sistema deduzca los resultados.

Los Hechos

Son verdades fundamentales que establecemos dentro de nuestro sistema lógico.

Ejemplo en Prolog:

humano(socrates).
animal(perro).

Esto afirma que Sócrates es humano y el perro es un animal.

Las Reglas

Definen relaciones lógicas que permiten al sistema inferir nueva información a partir de los hechos.

Sintaxis conceptual:

"Si X es humano, entonces X es mortal."

mortal(X) :- humano(X).

Motor de Inferencia

Es el componente que procesa los hechos y las reglas para responder consultas.

  • Unificación: Busca coincidencias entre la consulta y la base de conocimientos.
  • Backtracking: Vuelve atrás si una ruta lógica no llega a una solución.

¿Dónde se aplica?

Inteligencia Artificial

Sistemas de diagnóstico médico y razonamiento automático.

Procesamiento de Lenguaje

Análisis gramatical y traducción automática de idiomas.

Bucle Repetir (Do-Until)

Es una estructura de control que repite un bloque de instrucciones hasta que se cumple una condición.

A diferencia del bucle "Mientras", el bloque de código se ejecuta al menos una vez, ya que la condición se evalúa al final.

Repetir vs Mientras

Mientras (While)

Evalúa antes de entrar. Puede ejecutarse 0 veces.

Repetir (Until)

Evalúa después de entrar. Se ejecuta mínimo 1 vez.

Sintaxis en Pseudocódigo

Repetir
   instrucciones...
Hasta Que condicion

El bucle termina cuando la condición es Verdadera.

Ejemplo: Validación de Datos

Pedir un número hasta que sea positivo:

Repetir
  Escribir "Ingrese un número mayor a 0:"
  Leer n
Hasta Que n > 0

Si el usuario ingresa -5, el programa volverá a preguntar.

¿Cuándo usar Repetir?

  • Menús de Usuario: Para mostrar las opciones y volver a pedirlas si la opción es inválida.
  • Ingreso de Contraseñas: Para asegurar que el código se ejecute al menos una vez antes de verificar.
  • Cualquier proceso que deba intentar una acción inicial antes de decidir si continúa.
