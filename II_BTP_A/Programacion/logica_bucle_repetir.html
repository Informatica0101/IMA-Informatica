Programación Lógica

Paradigma de programación basado en el uso de funciones matemáticas y lógica formal para resolver problemas complejos.

A diferencia de la programación imperativa, aquí se definen qué resultados se desean obtener en lugar de detallar cada paso del proceso.

Base de Conocimientos

En este paradigma, el programa se compone de un conjunto de hechos y reglas que describen el dominio del problema.

  • Hechos: Afirmaciones que son siempre verdaderas (ej. "Juan es padre de Pedro").
  • Reglas: Instrucciones condicionales (ej. "Si A es padre de B, entonces A es ancestro de B").

Motor de Inferencia

Es el "cerebro" del lenguaje lógico que busca respuestas dentro de la base de conocimientos usando algoritmos de búsqueda.

Utiliza el Backtracking: si una ruta de razonamiento falla, el motor regresa al último punto de decisión e intenta una alternativa.

Lenguaje Prolog

PROgrammation en LOGique es el lenguaje más representativo de este paradigma, nacido en los años 70 para procesar lenguaje natural.

Aplicaciones Actuales

Sistemas expertos, inteligencia artificial, diagnóstico médico y bases de datos deductivas.

Ideal para problemas donde las reglas son muy complejas y cambiantes, como la legislación o la genética.

Bucle Repetir (Do-Until)

Estructura de control que ejecuta un bloque de código antes de evaluar la condición de salida.

Garantía: El código dentro de un bucle Repetir siempre se ejecutará al menos una vez.

Sintaxis en PSeInt

   Leer clave
Hasta Que clave == 1234

Diferencia con "Mientras"

Mientras (While) evalúa al inicio; Repetir (Until) evalúa al final.

Mientras

Si la condición es falsa al inicio, nunca se ejecuta.

Repetir

Incluso si la condición es falsa, se ejecuta una vez inicialmente.

Condición de Salida

En PSeInt y muchos lenguajes, el bucle continúa mientras la condición sea Falsa y termina cuando es Verdadera.

Es fundamental asegurar que la condición se cumpla en algún momento para evitar bucles infinitos.

Casos de Uso Comunes

  • Menús interactivos: Mostrar opciones hasta que el usuario elija "Salir".
  • Validación de entradas: Pedir un dato hasta que cumpla con el formato correcto.
  • Procesos interactivos: Juegos que preguntan "¿Desea jugar de nuevo?".
