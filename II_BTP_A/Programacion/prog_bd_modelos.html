¿Qué es una Base de Datos (BD)?

Es un conjunto de información perteneciente a un mismo contexto, organizada sistemáticamente para su posterior recuperación, análisis y transmisión.

Ejemplo: El sistema de notas de la escuela, el inventario de una tienda.

El DBMS

DBMS (Database Management System): Es el software que permite a los usuarios interactuar con la base de datos (crear, leer, actualizar, borrar).

Ejemplos: MySQL, Oracle, SQL Server, MongoDB.

Modelos de Datos

Relacional (SQL)

Usa tablas con filas y columnas vinculadas por llaves.

No Relacional (NoSQL)

Usa documentos (JSON) o grafos. Muy flexible.

Estructura Relacional

  • Tabla: Colección de registros.
  • Fila (Registro/Tupla): Una unidad completa de información.
  • Columna (Campo/Atributo): Un tipo de dato específico (ej. Nombre).

Llaves Primarias (PK)

Es un campo único que identifica de forma irrepetible a cada registro en una tabla.

Ejemplo: El número de identidad o la matrícula de un auto.

Integridad de Datos

Asegura que la información sea exacta y consistente. Las reglas de integridad evitan, por ejemplo, que un alumno tenga una nota negativa.

Normalización

Es el proceso de organizar los datos para evitar la redundancia (repetición innecesaria) y asegurar que las relaciones tengan sentido lógico.

Relaciones

¿Cómo se conectan las tablas?

1 a 1 (Persona -> Pasaporte)
1 a Muchos (Cliente -> Facturas)
Muchos a Muchos (Estudiantes -> Clases)

El Lenguaje SQL

Es el estándar para interactuar con BD relacionales:

INSERT INTO alumnos (id, nombre, nota)

VALUES (101, 'Pablo', 95);

 

SELECT * FROM alumnos WHERE nota > 70;

Importancia

"En la era de la información, quien domina las bases de datos, domina el corazón de cualquier aplicación moderna."
