¿Qué es la Gestión de Proyectos?

Es la aplicación de conocimientos, habilidades y herramientas para liderar el trabajo de un equipo y lograr objetivos específicos del software dentro de los límites de tiempo, costo y calidad.

El Triángulo de Hierro

Todo proyecto se balancea entre tres factores:

  • Alcance: Lo que se va a construir.
  • Tiempo: Fecha de entrega.
  • Costo: El presupuesto disponible.

Si cambias uno, los otros dos se ven afectados.

Fases del Proyecto

1. Inicio
2. Planeación
3. Ejecución
4. Control
5. Cierre

Fase 1: Inicio

Se define la visión del proyecto y se obtiene la autorización formal. Se identifica al Project Manager y a los interesados (Stakeholders).

Fase 2: Planeación

Se crea la EDT (Estructura de Desglose de Trabajo), el cronograma y el plan de gestión de riesgos. Es la fase más importante para evitar el fracaso.

Fase 3: Ejecución

El equipo desarrolla el código según lo planeado. El PM gestiona a las personas y los recursos para que el trabajo fluya sin bloqueos.

Fase 4: Seguimiento y Control

Se mide el progreso real contra lo planeado. Si hay desviaciones, se toman acciones correctivas inmediatamente.

Fase 5: Cierre

Entrega final al cliente, liberación del equipo de trabajo y realización de la "Lecciones Aprendidas" para mejorar en el futuro.

Metodologías Ágiles

En lugar de una entrega final gigante, se hacen entregas pequeñas y funcionales cada 2 o 4 semanas (Sprints). Ejemplo: Scrum.

Gestión de Riesgos

Identificar qué podría salir mal (ej. renuncia de un programador, corte de luz) y tener un Plan B listo.

Herramientas de Gestión


  • Jira

  • Trello

  • MS Project

Hitos del Proyecto

Son puntos de control importantes que no tienen duración (ej. "Firma del contrato", "Versión Beta terminada"). Sirven para celebrar el progreso.
