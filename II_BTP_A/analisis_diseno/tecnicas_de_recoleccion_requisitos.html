¿Qué son los Requerimientos?

Condiciones o capacidades que un sistema debe poseer para satisfacer necesidades.

  • Son las necesidades y expectativas de los usuarios.
  • Fundamentales para el éxito de un proyecto de software.

Tipos de Requerimientos

Funcionales

  • Lo que el sistema debe hacer.
  • Ej: Registrar usuarios, generar reportes.

Proceso de Recolección

Pasos clave para obtener la información necesaria.

  1. Identificación de Stakeholders.
  2. Definición del Alcance.
  3. Selección de Técnicas.
  4. Ejecución y Documentación.
  5. Validación y Verificación.

Técnica 1: Entrevistas

Conversaciones directas para entender necesidades.

  • Pueden ser estructuradas o no estructuradas.
  • Ventaja: Información detallada, aclaración de dudas.
  • Desventaja: Consume tiempo, posible sesgo.

Técnica 2: Cuestionarios / Encuestas

Preguntas escritas para grandes grupos.

  • Útiles para información cuantitativa.
  • Ventaja: Eficiente para grandes grupos, anonimato.
  • Desventaja: No permite aclaraciones, baja tasa de respuesta.

Técnica 3: Observación

Observar a los usuarios en su entorno de trabajo.

  • Obtiene información de primera mano.
  • Identifica problemas no mencionados.
  • Desventaja: Puede ser intrusiva, efecto Hawthorne.

Técnica 4: Análisis de Documentos

Revisión de la documentación existente de la organización.

  • Fuente de información formal y objetiva.
  • Útil para entender el estado actual ("as-is").
  • Desventaja: Documentos desactualizados o incompletos.

Técnica 5: Prototipos

Versión simplificada del sistema para feedback temprano.

  • Clarifica requerimientos, reduce malentendidos.
  • Puede ser de baja o alta fidelidad.
  • Desventaja: Usuarios pueden esperar el sistema final.
