Análisis y Diseño de Sistemas

Undécimo Grado - Sección A

Docente: Pablo Antonio Peña Mancia

Fecha: 2026

¿Qué es un Sistema?

Es un conjunto de componentes que interactúan entre sí para lograr un objetivo común. En informática, nos enfocamos en los Sistemas de Información (SI).

Componentes de un SI

  • Hardware
  • Software
  • Personas
  • Datos

Análisis vs. Diseño

Análisis

¿QUÉ se necesita? Entender el problema y requisitos.

Diseño

¿CÓMO se hará? Crear el plano de la solución.

El Analista de Sistemas

Es el puente entre los usuarios (problema) y los programadores (solución). Debe ser un gran comunicador y tener visión de negocio.

Ciclo de Vida (SDLC)

Es el proceso que sigue un sistema desde que nace como idea hasta que se retira del uso. Consta de varias fases secuenciales y lógicas.

Importancia del Análisis

Detectar un error en la fase de análisis cuesta $1. Corregirlo cuando el sistema ya está programado puede costar hasta $1,000.

Recolección de Datos

¿Cómo sabemos qué construir?

  • Entrevistas
  • Encuestas
  • Observación

Diagramas de Flujo

Herramienta visual para representar la lógica de un proceso o el movimiento de los datos en el sistema.

Requisitos Funcionales

Son las acciones que el sistema DEBE realizar obligatoriamente.
Ejemplo: "El sistema debe permitir imprimir facturas."

Requisitos No Funcionales

Son las características de calidad, rendimiento y seguridad.
Ejemplo: "El sistema debe cargar en menos de 2 segundos."

Factibilidad

Determina si el proyecto se puede hacer considerando:

  • Técnica (Tecnología)
  • Económica (Dinero)
  • Operativa (Uso real)

El Prototipo

Es una versión preliminar que sirve para que el usuario valide el diseño antes de la construcción final.
