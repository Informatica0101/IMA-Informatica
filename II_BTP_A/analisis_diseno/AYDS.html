Docente: Pablo Antonio Peña Mancia

Fecha: 2026

¿Qué es un Sistema?

Es un conjunto de componentes que interactúan entre sí para lograr un objetivo común. En informática, nos enfocamos en los Sistemas de Información (SI).

Componentes de un SI

  • Hardware
  • Software
  • Personas
  • Datos

Análisis vs. Diseño

Análisis

¿QUÉ se necesita? Entender el problema y los requerimientos del cliente.

Diseño

¿CÓMO se va a hacer? Crear el plano técnico de la solución.

El Analista de Sistemas

Es el puente entre los usuarios (que tienen el problema) y los programadores (que crean la solución). Debe ser un gran comunicador y tener visión de negocio.

Ciclo de Vida (SDLC)

Es el proceso que sigue un sistema desde que nace como idea hasta que se retira del uso. Consta de varias fases secuenciales.

Importancia del Análisis

Detectar un error en la fase de análisis cuesta $1. Corregirlo cuando el sistema ya está programado puede costar hasta $1,000.

Recolección de Datos

¿Cómo sabemos qué construir?

  • Entrevistas
  • Encuestas
  • Observación

Herramientas Visuales

Los diagramas de flujo permiten representar visualmente la lógica de un proceso o el movimiento de los datos.

Requisitos Funcionales

Son las acciones que el sistema DEBE realizar.
Ejemplo: "El sistema debe permitir registrar nuevos clientes."

Requisitos No Funcionales

Son las características de calidad y rendimiento del sistema.
Ejemplo: "El sistema debe tener copias de seguridad automáticas cada 24 horas."

Estudio de Factibilidad

Determina si el proyecto se puede hacer considerando:

  • Técnica (Tecnología)
  • Económica (Dinero)
  • Operativa (Uso real)

El Prototipo

Es una versión preliminar del sistema que sirve para que el usuario valide el diseño sin necesidad de programar todo el código final.
