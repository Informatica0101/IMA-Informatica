¿Qué es el Modelado?

Es la creación de una representación gráfica de un sistema. Sirve para que analistas y clientes entiendan cómo funcionará el software antes de escribir una sola línea de código.

UML (Lenguaje Unificado)

UML (Unified Modeling Language) es el estándar mundial para documentar y diseñar sistemas de software mediante diagramas.

"Una imagen vale más que mil líneas de código."

Diagrama de Casos de Uso

Representa QUÉ hace el sistema desde el punto de vista del usuario (actor).

Componentes: Actores (monitos), Casos de Uso (óvalos) y Relaciones (líneas).

Diagrama de Clases

Representa la estructura estática del sistema. Muestra los objetos, sus atributos (datos) y sus métodos (acciones).

Diagrama de Secuencia

Muestra cómo interactúan los objetos entre sí a lo largo del tiempo. Es ideal para modelar la lógica de un proceso específico.

DFD (Flujo de Datos)

Describe cómo se mueven los datos a través del sistema, desde su entrada hasta su almacenamiento y salida final.

Diccionario de Datos

Es un catálogo que define detalladamente cada dato utilizado en los diagramas. Asegura que todos los desarrolladores usen los mismos nombres y tipos de datos.

Herramientas CASE

Programas que ayudan a crear estos diagramas de forma profesional.

  • Lucidchart
  • StarUML
  • Visual Paradigm

Beneficios del Modelado

  • Reduce errores en la programación.
  • Facilita el mantenimiento futuro.
  • Sirve como contrato visual con el cliente.

Modelado de Base de Datos

El Diagrama Entidad-Relación (DER) es fundamental para diseñar cómo se guardará la información en las tablas de una BD.

Prototipado

Es un "modelo rápido" del sistema (maqueta). Permite que el usuario vea cómo será la interfaz antes de que el backend esté listo.

Niveles de Abstracción

Los modelos pueden ser:

  • Conceptuales: Idea general (para el jefe).
  • Lógicos: Detalles técnicos (para el analista).
  • Físicos: Implementación real (para el programador).
