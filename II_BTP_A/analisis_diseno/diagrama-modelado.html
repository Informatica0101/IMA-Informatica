🟥 Modelo AS-IS (Actual)

Modelo actual con procesos manuales y alto riesgo de error.

Inicio
Recibe pedido por correo
📧 Requiere revisar el email manualmente. Si se pierde el correo, se pierde el pedido.
Anota en cuaderno
📝 Dato propenso a errores, ilegibilidad o extravío.
Revisa stock manualmente
📦 No hay sistema de inventario; esto consume tiempo y no es confiable.
¿Hay stock?
❓ Decisión basada en revisión visual o intuición. Riesgo de errores.
Envía mensaje al proveedor
📲 Comunicación no estructurada, sin trazabilidad ni registro automático.
Espera confirmación
⏳ Sin control del tiempo de espera ni forma clara de saber si fue recibido.
Anota confirmación en el cuaderno
🖋️ Repetición de pasos manuales: no garantiza consistencia.
Notifica al cliente
📤 Envío manual, sin plantilla ni control del estado del pedido.
Fin

🟩 Modelo TO-BE (Propuesto)

Flujo automatizado, más ágil y con menor riesgo de error.

Inicio
Pedido recibido automáticamente en sistema web
✅ Sistema registra el pedido y lo guarda con ID único.
Sistema revisa stock en tiempo real
✅ El inventario se actualiza con cada venta. Menos errores.
¿Hay stock?
✅ Decisión programada. Si no hay stock, se bloquea la venta o se espera reposición.
Sistema genera orden automática al proveedor
✅ Ahorra tiempo. Se puede integrar vía API o email automático.
Proveedor actualiza estado del pedido en sistema
✅ Mayor trazabilidad. El cliente puede ver el estado actualizado.
Sistema notifica al cliente automáticamente
✅ Comunicación estandarizada con correo o mensaje automatizado.
Fin

🟦 Modelo TO-DO (Acciones necesarias para implementar TO-BE)

Pasos técnicos y organizativos para la transformación digital.

✔️ Desarrollar sistema web para pedidos
Incluye carrito, formulario y base de datos.
✔️ Implementar inventario digital
Stock sincronizado, con alertas y control automático.
✔️ Automatizar orden al proveedor
Via email automatizado o API según capacidad del proveedor.
✔️ Panel de estado de pedidos para clientes
Interfaz amigable para rastrear pedidos.
✔️ Capacitación y pruebas
Dueño y personal deben aprender a usar el nuevo sistema.
