¿Qué es el Modelado?

Es la creación de una representación gráfica de un sistema. Sirve para que analistas y clientes entiendan cómo funcionará el software antes de escribir una sola línea de código.

UML (Lenguaje Unificado)

UML (Unified Modeling Language) es el estándar mundial para documentar y diseñar sistemas de software mediante diagramas estandarizados.

"Una imagen vale más que mil líneas de código."

Diagrama de Casos de Uso

Representa QUÉ hace el sistema desde el punto de vista del usuario (actor).

Componentes: Actores (monitos), Casos de Uso (óvalos) y Relaciones (líneas).

Diagrama de Clases

Representa la estructura estática del sistema. Muestra los objetos, sus atributos (datos) y sus métodos (acciones).

Diagrama de Secuencia

Muestra cómo interactúan los objetos entre sí a lo largo del tiempo. Es ideal para modelar la lógica de un proceso específico paso a paso.

DFD (Flujo de Datos)

Describe cómo se mueven los datos a través del sistema, desde su entrada hasta su almacenamiento y salida final.

Diccionario de Datos

Es un catálogo que define detalladamente cada dato utilizado en los diagramas. Asegura que todos los desarrolladores usen los mismos nombres.

Herramientas CASE

Programas que ayudan a crear estos diagramas de forma profesional:

  • Lucidchart
  • StarUML
  • Paradigm

Beneficios del Modelado

  • Reduce errores en la fase de programación.
  • Facilita el mantenimiento y actualización futura.
  • Sirve como contrato visual entre analista y cliente.

Modelado de Base de Datos

El Diagrama Entidad-Relación (DER) es fundamental para diseñar cómo se guardará la información en tablas.

Prototipado

Es un "modelo rápido" del sistema (maqueta). Permite validar la interfaz de usuario antes de la construcción real.

Niveles de Abstracción

  • Conceptual: Idea general de negocio.
  • Lógico: Detalles de atributos y relaciones.
  • Físico: Implementación real en el motor.
