Arquitectura de un S.O.

Un sistema operativo no es un bloque sólido, sino que se divide en capas para gestionar la complejidad.

EL NÚCLEO (KERNEL)

Es la parte central que interactúa directamente con el hardware (CPU, Memoria).

EL PROYECTIL (SHELL)

Es la interfaz que permite al usuario interactuar con el Kernel (ya sea por comandos o gráfica).

¿Qué es un Proceso?

Un proceso es un programa en ejecución. El S.O. decide qué proceso usa el CPU y por cuánto tiempo.

Estados de un proceso:

  • Nuevo: Se está creando.
  • Listo: Esperando turno del CPU.
  • Ejecución: Usando el procesador.
  • Bloqueado: Esperando un recurso (ej: teclado).

Administración de Memoria RAM

El S.O. debe llevar el control de qué partes de la memoria están siendo utilizadas y por quién.

Memoria Virtual:

Técnica que permite al S.O. usar espacio en el disco duro como si fuera RAM cuando esta se agota, permitiendo abrir más programas.

Sistemas de Archivos

Es la forma en que el S.O. organiza y estructura los datos en el disco duro para que puedan ser encontrados.

NTFS

Sistema estándar de Windows actual.

FAT32

Usado en memorias USB por su compatibilidad.

EXT4

Sistema de archivos principal de Linux.

APFS

Sistema moderno de Apple.

Sistemas Multitarea

Permiten que varios procesos o aplicaciones se ejecuten aparentemente al mismo tiempo, repartiendo ciclos de CPU a gran velocidad.

CONCURRENCIA

Capacidad de gestionar múltiples tareas activas.

Gestión de Usuarios

Monousuario

Solo un usuario puede interactuar con el sistema a la vez (ej: MS-DOS).

Multiusuario

Varios usuarios pueden acceder a los recursos simultáneamente (ej: Windows Server, Linux).

Drivers (Controladores)

Son programas pequeños que enseñan al Sistema Operativo cómo comunicarse con un hardware específico.

"Sin el driver adecuado, el S.O. no sabe que existe la impresora o la tarjeta de video".

CLI: El poder del texto

Aunque las ventanas son cómodas, los profesionales usan la consola para tareas de administración avanzada.

C:\Users\Pablo> dir
C:\Users\Pablo> ipconfig /all
C:\Users\Pablo> systeminfo

Software Libre vs Propietario

Propietario

Derechos reservados, pago por licencia. (Windows, macOS).

Libre / Open Source

Código abierto, libertad de uso y modificación. (Linux, Android).

Permisos de Archivo

El S.O. protege los datos asignando niveles de acceso a cada usuario sobre los archivos:

  • Lectura: Solo ver el contenido.
  • Escritura: Modificar o borrar.
  • Ejecución: Correr el programa.

El Administrador de Tareas

¿Qué nos dice?

Muestra el porcentaje de uso de: CPU, RAM, Disco y Red. Es la herramienta principal para diagnosticar por qué una computadora está lenta.

Actividad en clase

Instrucciones:

Abre el Símbolo del Sistema (CMD) en tu computadora y ejecuta el comando systeminfo. Escribe en tu cuaderno: El nombre del Sistema Operativo, la versión, el fabricante y la cantidad de memoria física instalada.

Criterio: Obtención correcta de los datos técnicos.
