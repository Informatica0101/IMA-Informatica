==================================================
REPORTE DE AUDITORÍA TÉCNICA – pseudocodigo.html
==================================================

CAPACIDAD EJECUTADA:
- Secuencial: FUNCIONA (Asignaciones y operaciones básicas correctas)
- Condicionales: FUNCIONA (Soporta Si-Entonces-Sino y anidamiento)
- Ciclos: FUNCIONA (Soporta Mientras, Para y Repetir)
- Funciones: FUNCIONA (Con limitaciones críticas en procedimientos)
- Arreglos: FUNCIONA (Soporta 1D y 2D, base 1)

DETECCIÓN DE ERRORES:
- Sintaxis: DEFICIENTE (Catch-all en Parser permite ejecución parcial errática)
- Léxico: DEFICIENTE (Omite símbolos desconocidos silenciosamente)
- Semántica: CORRECTA (Detecta división por cero y variables no definidas)

==================================================
HALLAZGOS CRÍTICOS
==================================================

1. CONFLICTO DE PALABRAS RESERVADAS (BUG CRÍTICO):
   Variables de una sola letra como 'y' (operador lógico AND) u 'o' (operador lógico OR)
   son tokenizadas como KEYWORD, lo que rompe cualquier algoritmo que use estas letras
   como identificadores (ej. y <- 10 falla).

2. PROCEDIMIENTOS NO SOPORTADOS:
   El intérprete solo permite llamadas a funciones dentro de expresiones o asignaciones.
   Una llamada independiente como 'Saludar("Jules")' genera un error de sintaxis
   porque el Parser no reconoce una llamada a función como una sentencia válida por sí misma.

3. MANEJO DE ERRORES PERMISIVO:
   El Parser utiliza bloques try-catch internos en 'parseStatement' que capturan errores,
   los imprimen en consola y saltan al siguiente token. Esto puede resultar en una
   "ejecución exitosa" de un programa que tiene errores de sintaxis graves en medio,
   ejecutando solo las partes que logró parsear.

4. SENSIBILIDAD A MAYÚSCULAS/MINÚSCULAS:
   Mientras que las palabras reservadas son insensibles (ALGORITMO == algoritmo),
   los identificadores (variables) SON sensibles. 'Variable' y 'variable' son
   entidades distintas, lo cual es inconsistente con el estándar PSeInt.

5. ASIGNACIÓN SILENTE EN ARREGLOS:
   La asignación a índices de arreglos fuera de rango no arroja error durante
   la ejecución, fallando silenciosamente o creando propiedades inesperadas en
   el objeto JavaScript subyacente. El error solo se lanza al intentar LEER
   un índice fuera de rango.

==================================================
VULNERABILIDADES
==================================================

- EXPOSICIÓN DE EVAL(): NO. El intérprete utiliza su propio motor de ejecución
  basado en un AST simplificado y no utiliza eval() para ejecutar el pseudocódigo.
- INYECCIÓN: El motor es puramente basado en JavaScript sin acceso directo al
  DOM o APIs del sistema desde el código ejecutado, limitando el impacto de
  código malicioso.
- BUCLES INFINITOS: No hay mecanismos de interrupción (timeout) en bucles,
  lo que permite colgar la pestaña del navegador con un simple 'Mientras Verdadero Hacer'.

==================================================
EJEMPLOS USADOS EN PRUEBAS
==================================================

- Test de conflicto 'y':
  Algoritmo Test
    y <- 10 // FALLA: 'y' se detecta como operador
  FinAlgoritmo

- Test de procedimientos:
  SubProceso Saludar(n)
    Escribir n
  FinSubProceso
  Algoritmo Test
    Saludar("Test") // FALLA: Sentencia no reconocida
  FinAlgoritmo

- Test de Mayúsculas:
  Algoritmo Test
    A <- 1
    Escribir a // ERROR: Variable no definida
  FinAlgoritmo
