El Concepto del Iceberg

Una aplicación web es como un iceberg. El Frontend es la parte que sobresale del agua y todos ven. El Backend es la gigantesca base sumergida que sostiene todo el peso pero no es visible.

¿Qué es el Frontend?

Es el "lado del cliente". Todo lo que el usuario ve y con lo que interactúa: botones, colores, fuentes, animaciones y menús.

Tecnologías base: HTML, CSS, JavaScript.

¿Qué es el Backend?

Es el "lado del servidor". Se encarga de la lógica, el procesamiento de datos, la seguridad y la comunicación con la base de datos.

Tecnologías: Node.js, Python, PHP, Java, SQL.

La Base de Datos (BD)

Es donde se guarda la información de forma permanente. El backend lee y escribe en ella para mostrarle al frontend lo que el usuario necesita.

¿Cómo se comunican?

Usan una API (Application Programming Interface). El frontend hace una petición (Request) y el backend envía una respuesta (Response), generalmente en formato JSON.

Ejercicio: Identificación

¿A qué lado pertenece cada tarea?

  • Validar que la contraseña sea correcta en la BD.
  • Cambiar el color de un botón a rojo al hacer hover.
  • Enviar un correo electrónico de bienvenida.
  • Mostrar una galería de fotos.

El Desarrollador Fullstack

Es el profesional que domina ambas áreas (Front y Back). Es capaz de construir una aplicación completa desde la interfaz hasta el servidor.

Frameworks de Frontend

Herramientas que facilitan el trabajo de diseño y lógica en el navegador:


  • React

  • Vue

  • Angular

Frameworks de Backend

Ayudan a estructurar el código del servidor de forma segura:

  • Express
  • Django
  • Laravel

La Nube (Cloud)

Los servicios como AWS, Google Cloud o Azure permiten hospedar tanto el Frontend como el Backend en servidores remotos accesibles desde todo el mundo.

Seguridad

La seguridad principal ocurre en el Backend. Nunca debemos confiar en el Frontend para procesos críticos, ya que el usuario puede ver y modificar el código del navegador.

Flujo de una App

  1. 1. Usuario hace clic en "Entrar" (Front).
  2. 2. Se envía usuario/clave por API al Back.
  3. 3. Backend verifica en la BD.
  4. 4. Backend responde "OK" o "ERROR".
  5. 5. Frontend muestra mensaje al usuario.
